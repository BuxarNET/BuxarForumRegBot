# Приоритеты (по убыванию важности)
1. **Безопасность** — валидация всех входных и промежуточных данных
2. **Стабильность** — не ломать существующую работающую логику
3. **Производительность** — минимизация потребления памяти и CPU
4. **Читаемость** — документация и понятный код

# Протокол Рефакторинга (КРИТИЧНО)
Перед изменением существующего кода:
1. Проанализируй функцию: назначение, вход/выход, побочные эффекты
2. Оцени риски: может ли изменение сломать работающую логику
3. **ЗАПРОСИ разрешение у пользователя:**
   - Опиши, что функция делает сейчас
   - Объясни, зачем ты хочешь её изменить
   - Опиши потенциальные риски
   - Жди подтверждения перед применением изменений
